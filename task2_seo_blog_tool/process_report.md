# 📑 Task 2 Execution Process Report: SEO Blog Creator & Publisher

This execution report details the inputs, step-by-step technical processing pipeline, and final output files generated by the SEO Blog Post Creation and Publishing Tool.

---

## 📥 1. Pipeline Inputs

The execution pipeline takes a base product search term as input:
*   **Product Input Keyword:** `"wireless earbuds"`
*   **Target Publishing Channels:** `local` (saves markdown file to workspace) and/or `devto` (publishes live via Dev.to Web API)

---

## ⚙️ 2. Step-by-Step Processing

Once triggered, the tool executes four sequential automated processes:

### Step 2.1: E-Commerce Product Scraping
*   **Process:** The system crawls eBay's active search listings for the selected product keyword.
*   **Action:**
    1.  Formulates a request query URL: `https://www.ebay.com/sch/i.html?_nkw=wireless+earbuds`
    2.  Flashes a GET request pretending to be a modern web browser to avoid bot blocks.
    3.  Parses the returning HTML body using `BeautifulSoup4`.
    4.  Extracts the top 3 best-selling or trending item listings, compiling their **Title**, **Price**, and **Product Link**.

### Step 2.2: Autocomplete Keyword Sourcing
*   **Process:** Conducts organic keyword suggestion harvesting to locate trending search phrases.
*   **Action:**
    1.  Sends a request to Google's suggestion endpoint: `http://suggestqueries.google.com/complete/search?client=firefox&q=wireless+earbuds`
    2.  Extracts the raw JSON array.
    3.  Filters and returns the top 3-4 highly relevant long-tail search queries (e.g. `wireless earbuds under 50`, `wireless earbuds under 100`, `best wireless earbuds`, `buy wireless earbuds`).

### Step 2.3: Gemini Blog Post Generation
*   **Process:** Formulates a tailored prompt incorporating the scraped product catalog and target long-tail SEO phrases.
*   **Action:**
    1.  Assembles a prompt requesting a clean, engaging 150-200 word product review.
    2.  Calls the Google Gemini API (`gemini-3.5-flash` / fallback models).
    3.  Instructs the model to naturally weave the exact keyword phrases into the article's text, rendering it in structured Markdown format.

### Step 2.4: Automated Publishing
*   **Process:** Saves the file and pushes it to external blogging platforms.
*   **Action:**
    1.  Saves a copy of the post as a `.md` markdown file in the `task2_seo_blog_tool/blogs/` directory.
    2.  If the publishing mode is `devto` or `both`, it reads the `.env` file's `DEVTO_API_KEY`, posts the markdown body payload using JSON to the Dev.to endpoint (`https://dev.to/api/articles`), and extracts the resulting live article web link.
