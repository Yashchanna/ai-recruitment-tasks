# 📑 Task 1 Execution Process Report: AI Video Generation

This execution report details the inputs, step-by-step technical processing pipeline, and final compiled output files generated by the AI Video Generation Tool.

---

## 📥 1. Pipeline Inputs

The execution pipeline takes two forms of input:
1.  **Manual Topic (CLI Override):** A custom topic string provided by the user (e.g. `--topic "Self-driving Cars Revolution"`).
2.  **Scraped Topic (Default RSS Mode):** If no manual topic is provided, the tool scrapes the **Google News RSS Feed** (`https://news.google.com/rss`) and selects the top trending story as the basis for the video.

### Example Run Input
*   **Topic:** `"Self-driving Cars Revolution"`
*   **RSS Source Context:** `"Self-driving cars are trending as major cities deploy robotaxi fleets globally, raising safety and technological questions."`
*   **Edge TTS Voice Choice:** `"en-US-JennyNeural" (Female - Conversational)`

---

## ⚙️ 2. Step-by-Step Processing

Once the input is acquired, the tool routes it through five distinct stages:

### Step 2.1: Gemini AI Script Generation
The script generator packages the topic and context into a system prompt requesting exactly 4-5 scenes structured in a valid JSON schema.
*   **Process:** Queries `gemini-3.5-flash` or fallback models (e.g. `gemini-2.5-flash`, `gemini-2.0-flash`) over the Google Generative AI API using the `response_mime_type="application/json"` parameter to guarantee strict schema parsing.
*   **Action:** The API returns:
    *   Catchy video title.
    *   Spoken voiceover script per scene (approx. 15-20 words).
    *   Search terms to query matching stock images.
    *   Punchy overlay text strings to overlay on the video.

### Step 2.2: Text-to-Speech (TTS) Narration
*   **Process:** The narration text block for each scene is sent to the `edge-tts` engine.
*   **Action:** The system connects to Microsoft's neural TTS service, renders realistic human speech based on the selected voice model (`en-US-JennyNeural`), and saves the audio files as independent `.mp3` files in a temporary directory (e.g., `scene_1.mp3`).

### Step 2.3: Stock Asset Sourcing
*   **Process:** The search queries generated by Gemini are sent to the image asset fetcher.
*   **Action:**
    1.  Tries to connect to the Pexels Search API (if key is set).
    2.  If the Pexels API is not set, it scrapes Unsplash search pages (`https://unsplash.com/s/photos/...`) for the first matching photo URL.
    3.  Downloads the high-resolution files locally as `scene_X.jpg`.

### Step 2.4: Pillow-Powered Caption Rendering
*   **Process:** Rather than relying on ImageMagick (which is highly unstable on Windows systems), the tool processes frames natively in memory.
*   **Action:**
    1.  Loads `scene_X.jpg` and crops it from the center into a vertical portrait ratio (1080x1920, 9:16).
    2.  Draws a semi-transparent slate card rectangle at the bottom (y=1500 to y=1750).
    3.  Wraps the scene's `overlay_text` and draws bold white sans-serif text onto the image frame.
    4.  Saves the final frame as a temporary annotated image.

### Step 2.5: MoviePy Compilation & Syncing
*   **Process:** The clips are loaded into MoviePy v2.x.
*   **Action:**
    1.  Reads the duration of the scene's narration audio clip.
    2.  Creates an `ImageClip` from the annotated image frame set to the exact duration of the audio.
    3.  Synthesizes the audio track into the image clip via `.with_audio()`.
    4.  Concatenates all scene clips sequentially using `concatenate_videoclips`.
    5.  Exports the completed project using the `libx264` video codec and `aac` audio codec.
    6.  Automatically deletes all temporary scene assets and folders.
